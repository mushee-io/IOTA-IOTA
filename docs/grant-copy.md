# Grant Copy – IOTA Builder Passport

## One-line summary

IOTA Builder Passport is an open-source trust and reputation layer for grants, hackathons, workshops, and ecosystem programs, giving builders portable credentials and giving committees a cleaner way to verify shipped work.

## Problem

Most grant and hackathon reviews are messy. Evidence lives across GitHub, docs, demo links, wallets, chats, and scattered milestone notes. Reviewers waste time piecing things together, while builders repeat the same proof every time they apply somewhere new.

## Solution

IOTA Builder Passport turns builder history into reusable trust infrastructure.

A builder creates a passport, links work, receives verifiable credentials for what they have shipped, and packages milestone evidence into notarized proof bundles. Committees can review a cleaner surface, issue recognition, and unlock stronger access based on trust state rather than raw wallet presence alone.

## Core flows

1. Create a builder passport
2. Issue a verifiable credential
3. Notarize a milestone proof bundle
4. Review the submission in a committee desk
5. Publish portable reputation
6. Unlock mainnet product surfaces with verified trust status

## Why this fits IOTA

This concept is shaped as an ecosystem tool, not just another generic dApp. It is meant to support public-good workflows around builders, open-source delivery, workshops, and events. It is also easy to understand, low risk, and suitable for open-source release.
